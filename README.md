# PriceParser

Консольная утилита для автоматического сбора цен из сервиса InfoPrice.by.
Забирает прайс-листы с штрихкодами, лезет за актуальными ценами и раскладывает результат по магазинам.

## Зачем это нужно

У некоторых торговых сетей в Беларуси нет открытых API для получения цен.
InfoPrice.by агрегирует цены по разным магазинам — остаётся только передать им
штрихкод и забрать то, что нужно. Эта программа делает всё это в автоматическом
режиме: кидаешь в папку excel-файлы, запускаешь, забираешь результат.

## Как это работает

1. Программа смотрит в папку `input` и собирает все excel-файлы.
2. В каждом файле ищет колонку с названием вроде "Штрихкод", "Штрих-код товара"
   или похожим. Проверяет все листы, все строки.
3. Для каждого найденного штрихкода делает POST-запрос к API InfoPrice.
   Если запрос упал по таймауту или сервер вернул 500 — повторяет ещё пару раз
   с небольшим интервалом.
4. Из JSON-ответа вытаскивает цены по магазинам: Корона, Гиппо, Грин, Евроопт,
   Санта, Соседи.
5. Результат пишет в новый excel-файл в папку `output`. Название формируется
   как `{имя_исходного_файла}-price-{дата}.xlsx`.
6. Сразу после сохранения результата строится мониторинговый отчёт —
   per-файл, данные передаются в память без переоткрытия Excel.
7. Обработанные исходники остаются на месте.

## Архитектура

Проект разделён на четыре слоя по принципу чистой архитектуры:

```
PriceParser/
├── input/          — сюда кладёшь excel-файлы со штрихкодами
├── output/         — сюда программа складывает результат
├── processed/      — (зарезервировано)
└── PriceParser.Console/
    ├── Program.cs              — точка входа, DI-контейнер
    │
    ├── Core/                   — Domain: модели и интерфейсы
    │   ├── Interfaces/         —   абстракции сервисов
    │   └── Models/             —   модельки данных
    │
    ├── Application/            — Use Cases: бизнес-логика, оркестрация
    │   ├── ParsingPipeline.cs  —   основной цикл обработки
    │   ├── PriceMappingService —   маппинг магазинов
    │   └── ValidationHelper    —   валидация конфига
    │
    ├── Infrastructure/         — реализация внешних сервисов
    │   ├── Excel/              —   ClosedXML: чтение, запись, мониторинг
    │   ├── Http/               —   HTTP-клиент для InfoPrice
    │   ├── Logging/            —   логирование в файл
    │   └── Parsing/            —   разбор JSON-ответа InfoPrice
    │
    ├── Configuration/          — настройки из appsettings.json
    └── Utils/                  — RetryHelper
```

### Зависимости между слоями

- **Core** — ни от кого не зависит
- **Application** — зависит только от `Core`
- **Infrastructure** — зависит от `Core` и частично от `Application`
- **Program.cs** — собирает всё через DI

## Настройка

В `appsettings.json` можно поменять:

- **InputFolder / OutputFolder / ProcessedFolder** — пути к папкам.
- **BarcodeColumnNames** — названия колонок, в которых искать штрихкоды.
  Программа ищет по вхождению подстроки, так что можно писать частично.
- **Monitoring / StoreMappings** — какие магазины отслеживать и как
  они называются в исходных файлах и в целевом отчёте.
- **RequestDelayMs** — задержка между запросами (чтобы не забанили).
- **TimeoutSeconds** — таймаут http-запроса.
- **RetryCount** — сколько раз перезапрашивать при ошибке.
- **MaxParallelism** — количество параллельных запросов к API.

## Чего тут нет

- Парсинга html-страниц — раньше был, но InfoPrice сделали json api,
  и html-парсер потерял актуальность. В коде не осталось.
- Визуального интерфейса — это консольная утилита.
- Поддержки других источников данных — только InfoPrice.

## Зависимости

- ClosedXML — работа с excel-файлами
- Microsoft.Extensions.Configuration.Json — конфигурация
- Microsoft.Extensions.DependencyInjection — di-контейнер
- .NET 10

## Как собрать и запустить

```sh
dotnet build
dotnet run --project PriceParser.Console
```

Либо скомпилировать self-contained exe и кинуть на сервер.
